# Mini App: Совместный список покупок / задач для Telegram

## Описание
Мини-приложение для Telegram, позволяющее создавать совместные списки задач или покупок прямо в чате.  
Все участники списка могут добавлять, редактировать и отмечать элементы как выполненные в реальном времени.

---

## Основные функции

### 1. Создание списка
- Возможность создать новый список с названием.
- Добавление участников прямо из чата.
- Назначение категории списка (опционально).

### 2. Элементы списка
- Название элемента (например, "Яблоки — 2 кг").
- Возможность указывать количество и приоритет.
- Отметка выполнения элемента.
- Назначение ответственного за выполнение (опционально).

### 3. Совместная работа
- Реальное обновление всех изменений для участников.
- История изменений: кто и когда добавил или отметил элемент.
- Фильтры по категориям или приоритету.

### 4. Напоминания
- Отправка уведомлений о невыполненных задачах.
- Настраиваемые частота и время напоминаний.

### 5. Экспорт и отчёты (Премиум)
- Экспорт списка в PDF или CSV.
- Сортировка и фильтрация перед экспортом.

---

## Интерфейс

### Главное окно
- Список элементов с чекбоксами.
- Кнопки:
  - `Добавить элемент`
  - `Удалить выполненные`
  - `Настройки`

### Меню “Ещё”
- Экспорт списка
- Фильтры
- Настройка напоминаний

### В чате
- Ссылка на список, открывающая мини‑app.
- Все участники видят актуальное состояние списка.

---

## Монетизация
- **Бесплатно:** базовый функционал списка и совместная работа.  
- **Премиум:**  
  - Неограниченное количество списков  
  - Экспорт в PDF/CSV  
  - Расширенные напоминания и уведомления  
  - Кастомизация (цвета, иконки, категории)

- Подключение к Telegram Payments для подписки.

---

## Техническая реализация

### Backend
- Облачная база данных: Firebase, Supabase или собственный сервер.
- Хранение данных с синхронизацией в реальном времени.

### Frontend
- Telegram Mini App (WebApp) на JS/React или Vue.
- UI минималистичный, оптимизирован под мобильные устройства.

### API Telegram
- WebApp API для интерфейса и взаимодействия с пользователем.
- Callback API для обработки действий (добавление, редактирование, отметка выполнения).
